1. The Generator Function

                        
Add this pure function to your codebase. It has zero dependencies.

                        
/**
 * Generates a BiDi-secured CAPTCHA
 * @param {string} elementId - The ID of the container to render into
 * @returns {string} The secret code for validation
 */
function generateBidiAuth(elementId) {
    // 1. Generate Random Code (5 chars)
    const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
    let secret = '';
    for(let i=0; i<5; i++) secret += chars.charAt(Math.floor(Math.random()*chars.length));
    
    // 2. Obfuscate (BiDi Reversal + Noise)
    const atoms = Array.from(secret);
    let noisy = [];
    atoms.forEach(a => {
        noisy.push(a);
        // Add Zero Width Space noise (60% probability)
        if(Math.random() > 0.4) noisy.push('\u200B'); 
    });
    
    // 3. Structural Reversal
    const reversed = noisy.reverse();
    const mid = Math.floor(reversed.length / 2);
    const p1 = reversed.slice(0, mid).join('');
    const p2 = reversed.slice(mid).join('');

    // 4. Render with BiDi Override
    const el = document.getElementById(elementId);
    el.innerHTML = `<div style="unicode-bidi:bidi-override; direction:rtl; text-align:center; user-select:none;">
      <span style="direction:rtl">${p1}</span><span style="direction:rtl">${p2}</span>
    </div>`;
    
    return secret; 
}

                    


                    

                        
2. Implementation Strategy

                        

                            

                                Do
                                
    
                                    
  • Regenerate code after every attempt (success or fail).
    • 
                                    
  • Use server-side validation if possible (send secret hash).
    • 
                                    
  • Add rate limiting (e.g., max 5 attempts per minute).
    • 
                                

                            

                            

                                Don't
                                
    
                                    
  • Use CSS `display: none` for honeypots (bots detect this).
    • 
                                    
  • Rely solely on this for high-value targets (banking).
    • 
                                    
  • Forget audio fallback for production accessibility.
